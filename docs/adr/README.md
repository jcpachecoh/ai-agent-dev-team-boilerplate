# Architecture Decision Records (ADRs)

This folder contains Architecture Decision Records managed by the **Architecture Agent**.

## What is an ADR?

An Architecture Decision Record captures an important architectural decision, the context in which it was made, and its consequences. ADRs serve as a permanent log of *why* the system is designed the way it is.

## Naming Convention

```
ADR-NNN-<short-title>.md
```

Example: `ADR-001-use-postgresql-as-primary-database.md`

## ADR Template

```markdown
# ADR-NNN: <Title>

## Status
Proposed | Accepted | Deprecated | Superseded by ADR-NNN

## Date
YYYY-MM-DD

## Context
What is the problem or decision that needs to be made?
What forces are at play (technical, business, organizational)?

## Decision
What was decided? State the decision clearly.

## Consequences
What are the positive and negative consequences of this decision?
What trade-offs were accepted?

## Alternatives Considered
What other options were evaluated and why were they not chosen?
```
